## 작성 규칙 요약

1. **확인된 것만 쓴다** — 코드, 커밋, 설정 파일에서 근거를 찾을 수 없으면 항목 자체를 생략한다.
2. **수치는 신중하게** — Git 통계처럼 자동 산출 가능한 것은 그대로. 운영 수치(DAU, 응답시간 등)는 플레이스홀더로 남긴다.
3. **섹션 유연성** — 근거 없는 섹션은 통째로 생략해도 된다. 빈 섹션보다 없는 섹션이 낫다.
4. **어조** — 기술 문서이므로 건조하고 명확하게. 마케팅 표현 ("혁신적인", "강력한") 사용 금지.
5. **길이** — 프로젝트 1개 기준 300~600줄이 적정. 너무 길면 트러블슈팅 항목 수를 줄인다.
